解决函数并发执行导致的数据丢失问题

🎉 根本问题已修复！

loadPolicyInfo 函数不再覆盖自定义字段数据

问题根因分析

核心问题： 多个函数并发执行，后执行的函数覆盖了先执行函数的数据。

问题执行时序：

T1
loadCustomFields() 执行完成
✅ 添加自定义字段到 policyInfo.value = {custom_xxx: {value: "1233435"}, ...}
T2
loadPolicyInfo() 执行
❌ 重置 policyInfo.value = {} ← 自定义字段数据丢失！
T3
visibleFields 计算
❌ 找不到自定义字段数据，页面显示空白

从控制台日志可以验证：

  • ✅ API请求成功: GET /api/v1/custom-fields/values/1
  • ✅ 数据返回正确: field_value: "1233435", "1234335366214"
  • ✅ loadCustomFields 执行正常
  • ❌ 但页面不显示，说明数据被后续函数覆盖了
❌ 修复前：数据覆盖
// loadPolicyInfo 函数 - 第1047行 if (basicPolicyData) { policyInfo.value = {} // ← 完全重置，丢失自定义字段 // 处理基本政策数据... Object.keys(basicPolicyData).forEach(fieldName => { policyInfo.value[fieldName] = { ... } }) }
结果： 自定义字段被清空，只剩基本政策字段
✅ 修复后：数据合并
// loadPolicyInfo 函数 - 修复后 if (basicPolicyData) { if (!policyInfo.value) { policyInfo.value = {} } // ← 仅在需要时初始化，保留现有数据 console.log('当前policyInfo包含:', Object.keys(policyInfo.value)) // 处理基本政策数据...（追加，不覆盖） Object.keys(basicPolicyData).forEach(fieldName => { policyInfo.value[fieldName] = { ... } }) }
结果： 保留自定义字段，添加基本政策字段
