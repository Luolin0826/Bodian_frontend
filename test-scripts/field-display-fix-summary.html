解决自定义字段值不显示的问题

🎉 修复完成！

自定义字段现在能正确显示字段值了

问题分析

根本原因： 前端代码只处理了字段定义（添加到 allFields），但没有处理字段值。

API返回的正确数据结构：

{ "data": { "fields": [ { "field_name": "custom_1757226925033", "display_name": "发展前景如何", "field_value": "1", // ← 这个值没有被前端使用 "field_type": "text", "is_visible": 1, ... } ] } }

问题表现：

  • ❌ 字段标题显示正常（"发展前景如何"、"福利待遇"）
  • ❌ 字段值不显示（应该显示 "1"、"2" 等值）
  • ❌ 界面上只能看到字段名，没有内容
❌ 修复前的代码逻辑
// 只处理字段定义，没有处理字段值 result.fields.forEach((field: any) => { if (field.is_visible) { allFields[field.field_name] = { display_name: field.display_name, type: field.field_type || 'text', description: field.field_content || field.display_name, is_custom: true } // 缺少：没有将 field_value 添加到 policyInfo } })
结果： 字段定义有了，但没有值
✅ 修复后的代码逻辑
// 同时处理字段定义和字段值 result.fields.forEach((field: any) => { if (field.is_visible) { // 1. 添加字段定义到allFields allFields[field.field_name] = { display_name: field.display_name, type: field.field_type || 'text', description: field.field_content || field.display_name, is_custom: true } // 2. 添加字段值到policyInfo policyInfo.value[field.field_name] = { value: field.field_value || '', display_name: field.display_name, type: field.field_type || 'text', priority: 999, data_source: 'custom_fields' } } })
结果： 字段定义和值都有了

修复方案详解

数据流程修复：

  1. API响应 → 包含字段定义和值的完整数据
  2. 字段定义处理 → 添加到 allFields（用于字段配置）
  3. 字段值处理 → 添加到 policyInfo.value（用于显示）
  4. 界面渲染 → 从 visibleFields 获取完整的字段信息显示

关键修复点：

  • ✅ 初始化 policyInfo.value = {} 确保对象存在
  • ✅ 将 field.field_value 映射到 policyInfo.value[fieldName].value
  • ✅ 保持数据结构一致性，符合现有显示逻辑
  • ✅ 添加详细日志，便于调试和监控
