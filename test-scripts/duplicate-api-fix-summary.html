发现的问题

重复请求原因：

  • 🔄 loadCustomFields() - 发送带 include_definitions=true 的请求
  • 🔄 loadBasicPolicyData() - 发送不带 include_definitions=true 的请求
  • 🔄 loadPolicyInfo() - 发送不带 include_definitions=true 的请求

导致的问题：

  • ❌ 同一个API被调用3次
  • ❌ 页面数据冲突，显示异常
  • ❌ 性能浪费，加载缓慢
  • ❌ 网络面板显示重复请求
❌ 修复前：3次重复请求
  • loadCustomFields()
    GET /api/v1/custom-fields/values/1?section=basic&province=广东省&include_definitions=true
  • loadBasicPolicyData()
    GET /api/v1/custom-fields/values/1?section=basic&province=广东省
  • loadPolicyInfo()
    GET /api/v1/custom-fields/values/1?section=basic&province=广东省
结果： 数据冲突，页面显示异常
✅ 修复后：1次统一请求
  • loadCustomFields()
    GET /api/v1/custom-fields/values/1?section=basic&province=广东省&include_definitions=true
  • loadBasicPolicyData()
    🚫 已删除重复调用
  • loadPolicyInfo()
    🚫 已删除重复调用
结果： 数据统一，页面正常显示

具体修复方案

1. 保留合并接口调用：

// loadCustomFields() - 保留 const result = await customFieldsAPI.getCustomFieldValues( props.unitId, section, provinceToUse, true // includeDefinitions )

2. 删除重复调用：

// loadBasicPolicyData() - 已删除 // const customFieldValues = await customFieldsAPI.getCustomFieldValues(unitId, 'basic', currentProvince.value) // loadPolicyInfo() - 已删除 // const customFieldValues = await customFieldsAPI.getCustomFieldValues(unitId, 'basic', currentProvince.value) // 替换为统一说明 console.log('✅ 自定义字段将由 loadCustomFields() 统一处理')
